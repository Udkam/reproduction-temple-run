# Design Options

## A. 量子实验室风

- Core fantasy: the player controls a quantum drone inside experimental puzzle chambers.
- Visual language: neon grid, glass panels, lab terminals, energy conduits, phase rings.
- Character: quantum drone with a bright core and modular fins.
- Mechanics fit: time echoes, portals, swaps, recursive chambers, and chain experiments all read naturally as lab protocols.
- Risk: can become sterile if too blue. Mitigation: use cyan/green/magenta accents with dark neutral surfaces.

## B. 深空飞船 / 星港风

- Core fantasy: repair probe navigating a starship or orbital harbor.
- Visual language: hull panels, airlocks, constellation map, warning lights.
- Character: maintenance robot or starship probe.
- Mechanics fit: portals as airlocks, chains as station power routing.
- Risk: can drift toward ordinary spaceship UI and weaker recursion metaphor.

## C. 赛博矩阵 / 数据空间风

- Core fantasy: cursor core moving blocks inside a data lattice.
- Visual language: data trails, scanlines, bitfields, terminal overlays.
- Character: data ghost or cursor core.
- Mechanics fit: time split and synchronized actors are easy to explain.
- Risk: one-note cyber palette and abstractness may make boxes/targets less readable.

## Choice

Choose A as the main direction. Borrow B's constellation chapter map and C's cursor/data feedback only as secondary language. The final product should read as Driftbox's own quantum puzzle lab, not as a copied reference.
