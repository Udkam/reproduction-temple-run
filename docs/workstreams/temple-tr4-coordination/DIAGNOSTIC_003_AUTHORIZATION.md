2. Repeat with original shift `1.0`; require original and evaluated shift within `1e-8` of `1.0`; calculate and retain the second full matrix.
3. Define `difference[i]=matrix1[i]-matrix0[i]` in Python row-major order. Require zero baseline `abs(matrix0[1][2])<=1e-7`. Require the maximum absolute off-target delta over all `i != 6` to be at most `5e-7`, and require `abs(response)/max(maxOffTargetCalibrationDelta,1e-12)>=1e6`. Require Python row/column `(1,2)`, row-major `6`, to map to Three/WebGL column-major `9` by `webglIndex=column*4+row`.
4. Solve `camera.data.shift_y=(lensShiftY-matrix0[1][2])/response`; reject non-finite values. Apply it with the same `update_tag -> view_layer.update -> fresh dependency graph/evaluated camera` sequence and calculate the final matrix.

Define `solvedDifference[i]=matrixSolved[i]-matrix0[i]`; every off-target `i != 6` must be at most `5e-7`. Convert the final Python matrix to Three/WebGL column-major order using `elements[column*4+row]=matrix[row][column]`; require index `9` target error and every other element's error against the frozen profile `projectionMatrix` to be at most `5e-7`.
