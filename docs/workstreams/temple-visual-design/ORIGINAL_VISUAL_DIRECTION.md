# TIDE//RELAY — Tide Scar / 潮痕追逐

## 冻结命题

这不是丛林、古庙或冒险电影的换皮。`TIDE//RELAY` 的路是一条晒褪的矿物脊道：风化沙面在深海蓝阴影上方被切开，危险只以珊瑚色暴露在断裂、摩擦和追逐的边缘。玩家是一名向前倾的短衣信使；追逐者是有足部接触、会改变距离的低潮双体，而不是动物、黑影或进度条。

唯一签名手势为 **潮痕线**：一条粉笔白、带少量沙色磨损的连续线。它只沿跑道右缘从远方进入近景，接近镜头时才因追逐压力收紧并短暂转为珊瑚；HUD 只保留 20 px（close chase 为 13 px）的短末端指示，不让线条穿越天空、暂停区或大面积画面。它是方向、速度和追逐距离的一个读法，不是科幻仪表。

## 构图与层级

1. 世界先读到：低对比断崖/信标远景、潮池或偏置石块中景、路缘碎屑与接地影近景，再读障碍、信使和追逐者。环境最多使用这三种轮廓，不形成丛林、神庙或装饰堆叠。
2. HUD 只读到：score、distance、flow、shards、暂停；全部值必须来自 canonical state，原型则来自同一 `mockState`。
3. 潮痕线比路缘和 HUD 都弱；其收紧只暗示 `chaseGap`，不能替代实体追逐者。
4. 250 m 只在世界的潮痕线上短暂出现，600–900 ms，不用中央弹窗。

## 明确禁止

- 神庙、丛林、宝物、猴/猿怪物、商业跑酷的角色/场景/镜头/布局复刻。
- 周期网格、点阵、青绿色系统发光、CRT、玻璃卡、圆角卡阵、右下浮层、巨型状态字、假坐标或假 telemetry。
- 装饰遮住真实碰撞体、用纯色区分危险、双脚滑行或无接地阴影的角色。
- 分数、米数、flow、shards 各自伪造或不一致；生产版必须直接从 canonical state 读值。
